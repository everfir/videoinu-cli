# 媒介原理速查

## 1. 资产本体

每种资产绑定的维度：

| 资产类型 | 绑定维度 |
|---------|---------|
| Reference Image | 身份 / 主体外观 / 风格基调 |
| First Frame | 精确起始画面 |
| Last Frame | 精确终止画面 |
| Motion Reference | 运动轨迹 / 节奏 / 相机运动 |
| Storyboard 线稿板 | 构图骨架 / 景别 / 运镜方向 |

Prompt 文字只补这些资产都没覆盖的维度。资产 > Prompt；同一张图不能当两种资产（比如不能同时做参考图和首帧）。

### 多资产叠加：看维度正交，不看数量

r2v / 多图 slot 下「传几张」不是靠「越少越稳」拍脑袋，按**维度是否正交**判断：

| 组合 | 维度 | 判断 |
|------|------|------|
| 主体 ref + storyboard | 身份 × 构图 | ✅ 正交，该传都传 |
| 主体 ref + 首帧 | 身份 × 起始画面 | ✅ 正交 |
| 主体 ref + motion ref | 身份 × 运动 | ✅ 正交 |
| 两张主体 ref（同一主体不同角度） | 身份 × 身份 | ❌ 重叠，特征稀释 |
| 两张风格 ref | 风格 × 风格 | ❌ 重叠 |
| 主体 ref + 风格 ref（同时锁身份和画风） | 身份 × 风格 | ⚠️ 边界情况：参考图的风格会渗到身份上，只在风格差异大时才传 |

**默认规则**：本任务跑过的上游资产（storyboard / 首帧）传给所有后续镜头，除非明确不需要这个维度约束（纯动作延续、i2v 只求动作、构图已由前一末帧锁定）。所有参考图传入时 prompt 里每个 `@imageN` 后都紧跟一句正面用途说明。

### 参考图复用

Agent 没有跨 session 记忆，用户说"用上次那个主体"时必须问清来源，不能猜。三种合法来源：

1. **本地文件**：让用户给文件路径
2. **历史 Plan**：用户指出 `plans/<prefix>.md` 里的某个已完成步骤
3. **已有资产引用**：用户直接给 CLI 层认识的资产标识

把来源原样写进 Plan 的 `input_spec`，如何解析成 CLI 能用的形式由 CLI skill 负责。

同一任务内的多镜头复用通过 Plan 步骤间的前序产物引用承载（见 plan-schema.md 与 CLI skill），不走这三条。

## 2. 生成模式决策树

| 模式 | 输入 | 何时选 |
|------|------|-------|
| t2v | 纯文本 | 没素材只有想法；风格化/梦境/概念类 |
| i2v | 图 + 文本 | 有主体参考但动作可变；产品宣传 |
| f2v | 首帧 + 文本 | 已确定开场画面；精确衔接前一镜头 |
| r2v | 参考 + 文本 | 跨视频复用某维度（用 `@image` 参数传入） |
| 首尾帧 | 首帧 + 末帧 + 文本 | 转场、变形、已知起止 |

选择顺序（从严到松）：首尾帧 → f2v → i2v/r2v → t2v

### 多镜头模式选择

- t2v 多镜头：一致性只靠锚点词，会漂移
- f2v 多镜头：上一末帧当下一首帧，衔接最稳
- i2v 多镜头：参考图锁身份，但镜头推进后画面与 ref 差距变大，需配合「多镜头参考图优化」动态更新 ref
- 不同镜头可用不同模式，按各自约束选择

### 多镜头参考图优化

固定参考图随镜头推进会与当前内容差距变大。默认的处理是用前序镜头的截图作为下一镜头的 ref；场景跳变大时把前序截图与主 ref 溶图；没有前序截图可对主 ref 做 image edit。简单场景维持原 ref 即可。

## 3. 一致性层级

| 层级 | 靠什么 | 断了的症状 |
|------|--------|-----------|
| 身份 | 参考图 | 主体外观变了 |
| 风格 | 风格锚点词 / 参考图 | 画风漂移 |
| 构图 | 首帧 / 末帧 | 镜头接不上 |
| 运动 | motion reference | 节奏乱 |
| 色彩/光照 | 参考图 / 首帧 | 色调跳 |

诊断顺序：身份 → 风格 → 构图 → 运动 → 色彩。

## 4. Storyboard 线稿图规格

触发条件见 SKILL.md「Storyboard」。本节只规定每张板的规格，**具体的提示词模板 + 可填槽位**见 [storyboard-template.md](storyboard-template.md)，别现编、别手搓。

### 核心原则

**决定跑 storyboard 的镜头各跑一张多宫格时序板**，Plan 里对应一个 `step_sb_N` 步骤。一张图内部按该镜头的主要时序节拍拆成 N 宫格，**一格一个时间步**，从左到右、从上到下阅读即是该镜头的完整动作时间轴。

- ❌ 一张大板覆盖多个镜头（挤格、邻镜头污染）
- ❌ 每镜头单格（没给模型看到时序演进）
- ✅ 每镜头一张多宫格时序板（格数 = 该 shot 状态转变链的节拍数）

### 规格（多宫格时序板）

用 image 工作流每次出**一张多宫格黑白线稿 storyboard**。参考形态：漫画分镜板 / 动画 storyboard，不是技术线框图。

| 参数 | 值 |
|------|---|
| 风格 | 黑白手绘线稿，可有粗细线变化，允许 **manga-style 速度线 / 冲击射线 / 破碎线 / 残影**等动势表达，不要阴影色块不要彩色不要纹理 |
| 布局 / 画幅 | 见 [storyboard-template.md §`{{GRID_LAYOUT}}` 选型](storyboard-template.md#grid_layout-选型) |
| 边框 | **粗黑矩形边框分隔每格**，纯白格内背景 |
| 输入 | 主体参考图按 `input_spec.images[]` 顺序绑定到 `@image1 / @image2 / ...`；每张的用途说明文本直接套 [storyboard-template.md](storyboard-template.md#subject_ref_block-格式) 的 `{{SUBJECT_REF_BLOCK}}`，不自己编 |
| 只锁 | 该镜头的时序骨架：每格的主体姿态动作、环境事件因果、动势表达；以及必要时手工覆盖的调度（破题/冲击/情绪关键格） |
| 不锁 | 纹理、光照、着色、主体细节、风格、身份（由正片参考图和 prompt 负责）；每格景别视角默认由模型按动作强度自动变化，不手工指定 |

**不要做的事**：
- ❌ 在图面上标「start / end / 运镜箭头 / 文字标签」——storyboard 的时序靠格子顺序和格内画面表达，不靠文字注释
- ❌ 在格间画"小箭头解释运镜"——同上
- ❌ 追求"clean structural thin line"式的技术线框感——太干净会失去 storyboard 的信息密度和动势表现力
- ❌ 现场手搓 prompt / 只粘贴 frame 不填 Panel → 见 storyboard-template.md 反模式段

**必须做的事**：
- ✅ 用 [storyboard-template.md](storyboard-template.md) 的标准模板，只填 slot，不改 frame
- ✅ prompt 里**显式逐格列出**（Panel 1: ... Panel 2: ...），模板的 `{{PANEL_BEATS}}` slot 必填
- ✅ 动势强的格（冲刺、劈斩、撞击）显式写出"速度线 / 冲击射线 / 残影模糊"
- ✅ 相邻格景别和视角要变化，不要六格都是中景
- ✅ 勾起读者看懂时序：前后格之间的动作要有因果关系（A 动作完成 → B 结果发生）

### 提示词模板

不要在本文件里维护两份模板。标准模板 + 所有可填 slot 的写法约束**只在** [storyboard-template.md](storyboard-template.md)。写 `step_sb_N` 的 prompt 时直接读那个文件，复制骨架、填 slot。

**节拍写法**：Panel 段落每一格一句，主语-动作-景别-视角-环境事件-动势标识。想让模型读懂"这是时序"，格子之间的因果关系要清楚（A 之后发生 B）。

### 回传给正片

每张 SB_N 跑出来后，作为自己那个正片镜头 `shot_N` 的构图 + 时序参考图传回去。依据见 §1「多资产叠加」（主体 ref × storyboard 是维度正交组合）。

`@image2` 传的是**当前镜头自己的 SB_N**，不是别的镜头的。顺序错位按 SKILL.md「身份核对」处理。正片 prompt 的权威链和骨架见 SKILL.md「有资产时的正片 prompt」。

## 5. 反直觉速查

- 加更多 prompt ≠ 更准确：超量后模型削权重
- 参考图越多 ≠ 越稳：**维度重叠时**多张会稀释特征（两张主体、两张风格）；**维度正交**组合（主体+构图、主体+首帧）不适用此条，见 §1 多资产叠加
- `4K, 8K` 只影响风格偏好，不影响输出分辨率
- Negative 用正向词描述不想要的失败项（`blurry` / `watermark`），不要写反义句式（`not blurry`）
- seed 一致 ≠ 结果一致：任何参数变了都会漂
- i2v 的参考图 ≠ 首帧：参考图只绑身份
