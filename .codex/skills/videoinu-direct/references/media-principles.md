# 媒介原理速查

回答"为什么这样做有效"的底层知识。当遇到规则没覆盖的情况、用户说不对但说不清哪里不对、要在多种生成模式间选择时来查。

## 1. 资产本体：每种输入绑定什么

| 资产类型 | 绑定维度 | 不绑定 |
|---------|---------|-------|
| **Reference Image** | 身份 / 主体外观 / 风格基调 | 动作 / 构图 / 镜头 |
| **First Frame** | 精确起始画面（主体+构图+光照+色彩） | 后续动作方向 |
| **Last Frame** | 精确终止画面 | 中间过程 |
| **Motion Reference** | 运动轨迹 / 节奏 / 相机运动 | 主体外观 / 风格 |
| **LoRA / 风格模型** | 风格签名 / 画风 | 主体 / 构图 |
| **Prompt（文本）** | 一切未被上面绑定的维度 | 已被资产锁定的维度不要重复 |
| **Seed** | 随机性一致（同 seed + 同 prompt 出同结果） | 任何语义维度 |

### 三条硬规则

1. **资产 > Prompt**：资产和 prompt 冲突时模型信资产。**不要用 prompt 描述参考图已锁定的维度**。
2. **维度不重叠的资产可叠加**：参考图（身份）+ motion ref（动作）+ LoRA（风格）可同时用。维度重叠会冲突。
3. **一张图不能同时当参考图和首帧**：前者绑身份允许构图变化，后者绑整个画面。

## 2. 生成模式决策树

| 模式 | 输入 | 牺牲 | 换来 | 何时选 |
|------|------|------|------|-------|
| **t2v** | 纯文本 | 可控性、一致性 | 最大自由度 | 没素材只有想法；风格化/梦境/概念类 |
| **i2v** | 图 + 文本 | 风格自由度 | 身份/主体稳定 | 有角色参考但动作可变；产品宣传 |
| **f2v** | 首帧 + 文本 | 起始自由度 | 开场精确可控 | 已确定开场画面；精确衔接前一镜头 |
| **r2v** | 参考 + 文本 | 取决于参考绑的维度 | 对应维度锁定 | 跨视频复用某维度（锚点优先用 `@image` 参数传入，文字锚点仅兜底） |
| **首尾帧** | 首帧 + 末帧 + 文本 | 中间过程自由 | 起止精确 | 转场、变形、已知起止让 AI 补中间 |

### 选择顺序（从严到松）

1. 有首帧 + 末帧约束 → 首尾帧
2. 有精确起始画面 → f2v
3. 有角色/主体需稳定但构图可变 → i2v / r2v
4. 有运动/运镜要复用 → r2v + motion reference
5. 什么都没有 → t2v

### 多镜头约束

- t2v 多镜头：每镜头独立，一致性只靠锚点词兜底，仍会漂移
- f2v 多镜头：上一镜头末帧当下一镜头首帧，衔接最稳
- i2v 多镜头：每镜头塞同一张参考图，身份稳定但构图/光照会变化

角色一致性要求高时选 f2v 链式或 i2v 同参考，不选 t2v。

### 多镜头混合模式

同一任务中不同镜头可用不同生成模式，每镜头按自身约束选择：

| 镜头需求 | 选什么 | 为什么 |
|---------|--------|-------|
| 首镜头，有精确起始画面 | f2v | 锁住开场 |
| 中间镜头，保角色但构图自由 | i2v / r2v | 身份稳定，动作灵活 |
| 转场镜头，已知起止画面 | 首尾帧 | 起止精确 |
| 纯环境/空镜 | t2v | 无一致性负担 |
| 复用前序运镜节奏 | r2v + motion ref | 锁运动维度 |

Plan 每步必须标明使用的模式。模式切换时注意一致性层交接——身份层靠参考图传递，构图层靠前序末帧传递。

### 多镜头参考图优化策略

固定参考图随镜头推进会与当前内容差距变大，导致模型在"忠于参考图"和"执行 prompt"之间拉扯：

| 策略 | 做法 | 适用 |
|------|------|------|
| **前序截图直接做 ref** | 从前序 shot 截取角色最清晰帧 | 前序质量高且场景连续 |
| **前序截图 → 溶图** | 前序关键帧与主 ref 做 image blend | 前序质量高但场景跳变大 |
| **主 ref → image edit** | 对主 ref 改姿态/表情/光照 | 没有前序截图或质量差 |
| **维持原 ref** | 不额外处理 | 简单场景 |

选择顺序：直接做 ref → 溶图 → image edit → 维持原 ref。溶图/edit 后脸部特征变化太大时宁可用原 ref。Plan 中标注 `[type: ref-optimize]`。

## 3. 一致性从哪里来

| 层级 | 靠什么 | 断了的症状 | 修法 |
|------|--------|-----------|------|
| **身份层** | 参考图 / LoRA / 同 seed | 脸变了、产品变了 | 加参考图或改 i2v/r2v |
| **风格层** | LoRA / 风格锚点词 / 同参考图 | 画风漂移、材质变了 | LoRA > 风格参考图 > 锚点词 |
| **构图层** | 首帧 / 末帧 | 镜头接不上、空间错乱 | 上一末帧当下一首帧 |
| **运动层** | motion reference | 运镜不一致、节奏乱 | 加 motion ref |
| **色彩/光照层** | 参考图 / 首帧 / prompt 光照词 | 色调跳、光线方向变 | 首帧锁最稳 |

**诊断顺序**：身份 → 风格 → 构图 → 运动 → 色彩。

**关键反直觉**：锚点词解决的是**识别**，不是**一致性**。两次生成都"有黑发红裙女子"≠"同一个女子"。真正的身份一致必须靠资产。

**r2v 锚点硬规则**：有参考图时通过 `@image` 参数语法传入 `input_spec` > 文字描述锚点。`@image` 参数让模型直接从像素提取特征，锚定力远强于文字。文字锚点仅在无可用资产时兜底。

## 4. 模式与任务匹配

| 任务 | 首选模式 | 理由 |
|------|---------|------|
| 产品宣传片 | i2v / r2v | 产品是身份维度，必须资产锁定 |
| 角色短剧 | f2v 链式 / i2v | 角色一致性硬需求 |
| 风景氛围片 | t2v / f2v | 无主体一致性负担 |
| 概念艺术动起来 | f2v | 起始画面必须精确 |
| 跳舞换人 | r2v（motion + 参考图） | 两个维度分别锁定 |
| 转场/变形 | 首尾帧 | 起止已知 |
| 漫画分镜 | f2v 链式 | 画面已设计好，只需动起来 |

## 5. 可生成性的物理边界

以下分级反映模型**当前能力上限**，不是选择倾向。按上限做决策——模型能力在持续提升，保守假设只会产出平庸内容。

**模型基本能力（默认使用，不需要额外风险考量）**：2 人互动 | 单人复杂动作 | 多人场景（3+）| 复杂运镜 | 多步动作序列 | 宠物/动物 | 快速物理碰撞/破碎 | 环境动态

**边界能力（准备 Plan B，但 Plan A 永远是直接尝试）**：精确文字（>3词/非英文/变化中）| 镜子/反射链/多层透明叠加 | 5+ 人同框精确互动 | 复杂手部精细动作（如弹奏乐器的逐指运动）

**基础级（稳定但容易无聊——仅用于过渡，不做主体）**：单主体缓慢动作 | 纯相机运动 | 氛围变化

好的 Plan 把表现力最强的内容放在主体镜头，基础级仅做节奏过渡。全程基础级 = 叙事平淡 = 失败。

## 6. 反直觉速查

- **加更多 prompt ≠ 更准确**：超量后模型削权重，不如把关键信息放进资产
- **参考图越多 ≠ 越稳**：多张平均会稀释特征
- **高清关键词 ≠ 高清输出**：`4K, 8K` 只影响风格偏好
- **Negative 写反义词会反噬**：`not blurry` 模型可能理解成 `blurry`，只写真实失败模式
- **seed 一致 ≠ 结果一致**：任何参数变了都会漂
- **i2v 的参考图 ≠ 首帧**：参考图只绑身份，视频第一帧和参考图通常不完全一样
