# 媒介原理速查

## 1. 资产本体

每种资产绑定的维度：

| 资产类型 | 绑定维度 |
|---------|---------|
| Reference Image | 身份 / 主体外观 / 风格基调 |
| 设定图（Subject Sheet） | 身份（多面板强化版：多角度 / 多状态 / 关键细节） |
| First Frame | 精确起始画面 |
| Last Frame | 精确终止画面 |
| Motion Reference | 运动轨迹 / 节奏 / 相机运动 |
| 分镜图（Storyboard） | 构图骨架 / 景别 / 运镜方向 / 时序骨架 |

Prompt 文字只补这些资产都没覆盖的维度。资产 > Prompt；同一张图不能当两种资产（比如不能同时做参考图和首帧）。

设定图和分镜图都是 image 产物，分别是 pre-production 的两个阶段，详见 §4「Pre-production 流水线」。「主体」范围见 SKILL.md「Pre-production 两层资产」。

### 多资产叠加：看维度正交，不看数量

r2v / 多图 slot 下「传几张」不是靠「越少越稳」拍脑袋，按**维度是否正交**判断：

| 组合 | 维度 | 判断 |
|------|------|------|
| 主体 ref + 分镜图 | 身份 × 构图 | ✅ 正交，该传都传 |
| 主体 ref + 首帧 | 身份 × 起始画面 | ✅ 正交 |
| 主体 ref + motion ref | 身份 × 运动 | ✅ 正交 |
| 两张主体 ref（同一主体不同角度） | 身份 × 身份 | ❌ 重叠，特征稀释 |
| 两张风格 ref | 风格 × 风格 | ❌ 重叠 |
| 主体 ref + 风格 ref（同时锁身份和画风） | 身份 × 风格 | ⚠️ 边界情况：参考图的风格会渗到身份上，只在风格差异大时才传 |

**默认规则**：本任务跑过的上游资产（分镜图 / 首帧）传给所有后续镜头，除非明确不需要这个维度约束（纯动作延续、i2v 只求动作、构图已由前一末帧锁定）。所有参考图传入时 prompt 里每个 `@imageN` 后都紧跟一句正面用途说明。

### 参考图复用

Agent 没有跨 session 记忆，用户说"用上次那个主体"时必须问清来源，不能猜。三种合法来源：

1. **本地文件**：让用户给文件路径
2. **历史 Plan**：用户指出 `plans/<prefix>.md` 里的某个已完成步骤
3. **已有资产引用**：用户直接给 CLI 层认识的资产标识

把来源原样写进 Plan 的 `input_spec`，如何解析成 CLI 能用的形式由 CLI skill 负责。

同一任务内的多镜头复用通过 Plan 步骤间的前序产物引用承载（见 plan-schema.md 与 CLI skill），不走这三条。

## 2. 生成模式决策树

| 模式 | 输入 | 何时选 |
|------|------|-------|
| t2v | 纯文本 | 没素材只有想法；风格化/梦境/概念类 |
| i2v | 图 + 文本 | 有主体参考但动作可变；产品宣传 |
| f2v | 首帧 + 文本 | 已确定开场画面；精确衔接前一镜头 |
| r2v | 参考 + 文本 | 跨视频复用某维度（用 `@image` 参数传入） |
| 首尾帧 | 首帧 + 末帧 + 文本 | 转场、变形、已知起止 |

选择顺序（从严到松）：首尾帧 → f2v → i2v/r2v → t2v

### 多镜头模式选择

- t2v 多镜头：一致性只靠锚点词，会漂移
- f2v 多镜头：上一末帧当下一首帧，衔接最稳
- i2v 多镜头：参考图锁身份，但镜头推进后画面与 ref 差距变大，需配合「多镜头参考图优化」动态更新 ref
- 不同镜头可用不同模式，按各自约束选择

### 多镜头参考图优化

固定参考图随镜头推进会与当前内容差距变大。默认的处理是用前序镜头的截图作为下一镜头的 ref；场景跳变大时把前序截图与主 ref 溶图；没有前序截图可对主 ref 做 image edit。简单场景维持原 ref 即可。

## 3. 一致性层级

| 层级 | 靠什么 | 断了的症状 |
|------|--------|-----------|
| 身份 | 参考图 | 主体外观变了 |
| 风格 | 风格锚点词 / 参考图 | 画风漂移 |
| 构图 | 首帧 / 末帧 | 镜头接不上 |
| 运动 | motion reference | 节奏乱 |
| 色彩/光照 | 参考图 / 首帧 | 色调跳 |

诊断顺序：身份 → 风格 → 构图 → 运动 → 色彩。

## 4. Pre-production 流水线：设定图 → 分镜图 → 正片

正片 video 的两张常用 image 前置资产。决策判据（何时做 / 何时不做）见 SKILL.md「Pre-production 两层资产」；本节规定做了之后的规格和协作链，两者都不是必做项，**决定做了就按本节的职责分工来，不要让一张图兼任两件事**。

| 阶段 | 资产 | 锁什么 | 形态 |
|------|------|--------|------|
| Step 1 | 设定图（Subject Sheet） | 主体身份 | 横向多面板参考图 |
| Step 2 | 分镜图（Storyboard） | 构图骨架 + 时序 | 多面板时序板 |
| Step 3 | 正片 video | 运动 / 质感 / 运镜执行 | i2v / f2v / r2v |

两者维度正交（身份 × 构图），同一正片镜头可以同时传入——组合判据见 §1 多资产叠加。

### 设定图规格（横向多面板主体参考图）

核心思路与分镜图同构——**一张图内多面板**锁一个维度，但锁的是**身份**而不是时序：

| 项 | 值 |
|---|---|
| 形态 | 横向多面板（character turnaround / model sheet / prop sheet / environment sheet），单行或 2×N 网格 |
| 面板数 | 主体信息量决定；角色常见 3-6 个（正面/侧面/背面 + 可选表情/服装变化），场景 2-4 个（关键机位视角），道具 2-4 个（多角度 + 关键细节特写） |
| 风格 | **与正片目标风格一致**（彩色写实就彩色写实，2D 动画就 2D），**不要用分镜图的黑白线稿**——身份锁的是外观材质色彩，线稿会丢信息 |
| 背景 | 纯中性色（白/灰），消除环境干扰；场景设定图例外，此时面板内部是场景本身，面板间背景保持统一 |
| 光照 | 平光 / 中性 studio light；避免强方向光把主体一半吞进阴影 |
| 一致性锚 | 所有面板的主体必须是同一主体同一版本——比例、配色、材质、服装细节必须逐面板一致 |

**不要做的事**：

- ❌ 在一张设定图里混放多个不同主体（角色 A 和角色 B 塞一张）——每个主体一张独立设定图
- ❌ 给设定图加叙事动作或情绪（"愤怒挥剑"）——设定图锁静态身份，动作留给分镜图
- ❌ 画面面板之间切换风格或画风——身份会被稀释
- ❌ 把设定图当首帧使用——它是 ref 不是 frame

**喂给下游**：设定图作为 r2v / r2i 的 `@image` 传入后续镜头的 `input_spec.images[]`，和分镜图并列。在 prompt 里用 `@imageN — [subject tag] subject reference — identity and appearance are defined by this sheet, do not substitute` 说明用途（身份标签规则见 SKILL.md「多图必写用途说明」）。模型会从多面板里综合采样身份特征，比单张 ref 更稳。

### 协作链（设定图 + 分镜图 + 正片的引用关系）

Step key 命名规则见 [plan-schema.md「Step key 命名约定」](plan-schema.md#step-key-命名约定)。典型链：

1. `step_cs_M`（可选）：生成主体 M 的设定图
2. `step_sb_N`（可选）：生成镜头 N 的分镜图；若镜头涉及主体 M 则把 `cs_M` 作为 `@image` 传入。**当设定图风格与分镜图风格差异大时**（典型：彩色写实设定图 + 黑白线稿分镜图）要在用途说明里强制降级：`@image1 — [subject M tag] reference for identity and body proportion only, render as black-and-white line-art, do not copy color or texture`。两者风格本来就一致时（例如正片是黑白漫画风，设定图和分镜图都是线稿）不用降级
3. `shot_N`：正片镜头 N 同时引用 `cs_M`（锁身份，彩色原貌）和 `sb_N`（锁构图 + 时序）

`step_sb_N` 和 `shot_N` 必须成对设计。`sb_N` 不是漂亮分镜图交付物，而是 `shot_N` 的可读输入；`shot_N` 也不是重新发明动作，而是消费 `sb_N` 后补镜头执行层。Plan 里记录两句话即可：`sb_N locks: ...` 与 `shot_N adds: ...`。

只有设定图没有分镜图、或只有分镜图没有设定图都是合法的；按镜头复杂度和主体复杂度分别决定。每张 `sb_N` 只传回自己对应的那个 `shot_N`，不要把别镜头的 sb 传过来——顺序错位按 SKILL.md「身份核对」处理。

## 5. 分镜图规格

触发条件见 SKILL.md「Pre-production 两层资产」。本节只规定每张板的规格，**具体的提示词模板 + 可填槽位**见 [storyboard-template.md](storyboard-template.md)，别现编、别手搓。

### 核心原则

**决定跑分镜图的镜头各跑一张多面板时序板**，Plan 里对应一个 `step_sb_N` 步骤。一张图内部按该镜头的主要时序节拍拆成 K 个 panel（K = 该 shot 状态转变链的关键状态数），**一格一个时间步**。阅读顺序由布局决定，必须在 prompt 里写清楚。一镜一板——不要合并多镜头到一张，也不要单格。

K 没有固定上限。3-6 常用于短动作板，7-12 适合复杂动作链，13-16 更像动作研究 contact sheet。超过 16 默认拆成多张 storyboard，除非用户明确要总览。

### 规格（多面板时序板）

用 image 工作流每次出**一张多面板黑白线稿分镜图**。参考形态：漫画分镜 / 动画 storyboard / 动作研究 contact sheet，不是技术线框图。

| 参数 | 值 |
|------|---|
| 风格 | 黑白手绘线稿，可有粗细线变化，允许 **manga-style 速度线 / 冲击射线 / 破碎线 / 残影**等动势表达，不要阴影色块不要彩色不要纹理 |
| 布局 / 画幅 | 见 [storyboard-template.md §`{{LAYOUT_SPEC}}` 选型](storyboard-template.md#layout_spec-选型)；regular / bento / strip / contact sheet 都可用 |
| 边框 | **粗黑矩形边框分隔每格**，纯白格内背景 |
| 输入 | 主体参考图按 `input_spec.images[]` 顺序绑定到 `@image1 / @image2 / ...`；每张的用途说明文本直接套 [storyboard-template.md](storyboard-template.md#subject_ref_block-格式) 的 `{{SUBJECT_REF_BLOCK}}`，不自己编 |
| 只锁 | 该镜头的时序骨架：每格的主体姿态动作、环境事件因果、动势表达；以及必要时手工覆盖的调度（破题/冲击/情绪关键格） |
| 不锁 | 纹理、光照、着色、主体细节、风格、身份（由正片参考图和 prompt 负责）；每格景别视角默认由模型按动作强度自动变化，不手工指定 |

**不要做的事**：
- ❌ 在图面上标「start / end / 运镜箭头 / 文字标签」——分镜图的时序靠格子顺序和格内画面表达，不靠文字注释
- ❌ 在格间画"小箭头解释运镜"——同上
- ❌ 追求"clean structural thin line"式的技术线框感——太干净会失去分镜图的信息密度和动势表现力
- ❌ 现场手搓 prompt / 只粘贴 frame 不填 Panel → 见 storyboard-template.md 反模式段

**必须做的事**：
- ✅ 用 [storyboard-template.md](storyboard-template.md) 的标准模板，只填 slot，不改 frame
- ✅ prompt 里**显式逐格列出**（Panel 1: ... Panel 2: ...），模板的 `{{BEAT_LIST}}` slot 必填
- ✅ 动势强的格（冲刺、劈斩、撞击）显式写出"速度线 / 冲击射线 / 残影模糊"
- ✅ 布局要服务于节拍权重：权重均匀用 regular，有主转折用 bento，单方向轨迹用 strip，高 K 动作研究用 contact sheet
- ✅ 勾起读者看懂时序：前后格之间的动作要有因果关系（A 动作完成 → B 结果发生）。因果的载体是**至少一个可见物理状态量在相邻格间发生变化**（偏航角、抓地状态、液面倾斜、重心位置、形变程度），不是换景别或换角度。机械/载具/流体主体尤其要把叙事动词翻译成物理量变化，细则见 [storyboard-template.md「状态差原则」](storyboard-template.md#状态差原则)

### 回传给正片

`sb_N` 作为 `shot_N` 的构图 + 时序参考图传入（详见 §4 协作链）。正片 prompt 的权威骨架见 SKILL.md「有资产时的正片 prompt」。

## 6. 反直觉速查

- 加更多 prompt ≠ 更准确：超量后模型削权重
- 参考图越多 ≠ 越稳：看维度正交还是重叠（见 §1 多资产叠加）
- `4K, 8K` 只影响风格偏好，不影响输出分辨率
- Negative 用正向词描述不想要的失败项（`blurry` / `watermark`），不要写反义句式（`not blurry`）
- seed 一致 ≠ 结果一致：任何参数变了都会漂
- i2v 的参考图 ≠ 首帧：参考图只绑身份

更多创作原则见 SKILL.md「核心原则」。
